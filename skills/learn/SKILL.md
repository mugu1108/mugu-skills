# /learn - パターン抽出コマンド

**説明**: セッション中に解決した非自明な問題を再利用可能なスキルとして保存する。

## 使用方法

```
/learn [パターンの説明（省略可）]
```

## 概要

非自明な問題解決パターン、ライブラリの癖、デバッグテクニック等を `~/.claude/skills/learned/` に保存する。保存されたパターンは将来のセッションで自動的に参照可能になる。

## 起動タイミング

- 非自明なバグを解決した直後
- ライブラリの癖やワークアラウンドを発見した時
- デバッグで有効なツール組み合わせを見つけた時
- プロジェクト固有のパターンを定着させたい時

## 抽出対象

1. **エラー解決パターン** — 何のエラーが起き、根本原因は何で、何で直ったか
2. **デバッグテクニック** — 自明でないデバッグ手順、ツールの組み合わせ
3. **ワークアラウンド** — ライブラリの癖、API制限、バージョン固有の問題
4. **プロジェクト固有パターン** — コードベースの慣習、アーキテクチャ判断

## 抽出しないもの

- タイポや単純な構文エラー
- 一時的な問題（API障害等）
- 自明な修正

## ワークフローステップ

### ステップ1: セッションレビュー

会話を振り返り、抽出可能なパターンを特定する。

### ステップ2: パターンのドラフト

以下の形式でスキルファイルをドラフトする:

```markdown
# [パターン名]

**抽出日:** YYYY-MM-DD
**コンテキスト:** [いつこのパターンが適用されるか]

## 問題
[このパターンが解決する具体的な問題]

## 解決策
[パターン・テクニック・ワークアラウンドの詳細]

## 例
[コード例があれば記載]

## 適用条件
[このパターンが有効な状況]
```

### ステップ3: ユーザー確認

ドラフトをユーザーに見せ、保存前に確認を取る。

### ステップ4: 保存

```bash
mkdir -p ~/.claude/skills/learned
```

`~/.claude/skills/learned/[pattern-name].md` に保存する。

## 使用例

### 例1: API のワークアラウンド

```
/learn freee API のトークンリフレッシュで 401 が返る時の対処
```

→ トークンリフレッシュのタイミングと再試行ロジックをスキルとして保存

### 例2: セッション中のパターンを自動検出

```
/learn
```

→ セッションを振り返り、保存すべきパターンを提案

## ベストプラクティス

1. 1パターン1ファイルで保存する
2. 「問題」と「解決策」は具体的に書く（「動かなかった」ではなく「XエラーがYの原因で発生」）
3. 再現可能なコード例を含める

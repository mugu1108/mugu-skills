# Time Tracker Agent - 時間追跡エージェント

## 概要

Time Trackerエージェントは、業務委託の作業時間を正確に記録し、管理する専門エージェントです。Slackを通じて作業の開始・終了を記録し、Supabaseにデータを保存します。

## 使用タイミング

以下の場合にTime Trackerエージェントを使用してください:

1. **作業時間の記録**
   - プロジェクト別の作業開始・終了
   - 日次・週次・月次の作業時間集計
   - 複数プロジェクトの並行管理

2. **作業状態の確認**
   - 現在の作業状態の確認
   - 今日の合計作業時間の確認
   - プロジェクト別の作業時間確認

3. **データ同期**
   - Supabaseへの自動保存
   - 作業ログの永続化

## Slackコマンド

### `/in [プロジェクト名]` - 作業開始

```
@timebot /in プロジェクトA
```

**処理内容**:
1. プロジェクト名の検証（登録済みプロジェクトか確認）
2. 現在作業中のセッションがないか確認
3. Supabaseに開始時刻を記録
4. Slackに開始通知を送信

**応答例**:
```
✅ 作業を開始しました
📁 プロジェクト: プロジェクトA
🕐 開始時刻: 09:30
```

### `/out` - 作業終了

```
@timebot /out
```

**処理内容**:
1. 現在作業中のセッションを取得
2. 終了時刻と経過時間を計算
3. Supabaseに終了時刻・経過時間を記録
4. 今日の合計作業時間を計算
5. Slackに終了通知を送信

**応答例**:
```
✅ 作業を終了しました
📁 プロジェクト: プロジェクトA
🕐 終了時刻: 12:00
⏱️ 今回の作業時間: 2時間30分
📊 本日の合計: 5時間15分
```

### `/status` - 状態確認

```
@timebot /status
```

**処理内容**:
1. 現在の作業状態を取得
2. 作業中の場合は経過時間を計算
3. 今日の合計作業時間を取得

**応答例（作業中）**:
```
🔵 作業中
📁 プロジェクト: プロジェクトA
🕐 開始時刻: 09:30
⏱️ 経過時間: 1時間45分
```

**応答例（作業なし）**:
```
⚪ 作業していません
📊 本日の合計: 2時間45分
```

### `/today` - 今日のプロジェクト別内訳

```
@timebot /today
```

**処理内容**:
1. 今日の作業ログをプロジェクト別に集計
2. 各プロジェクトの作業時間と合計を表示

**応答例**:
```
📊 【今日の作業内訳】

📁 saixaid: 3時間30分
📁 プロジェクトB: 1時間0分

━━━━━━━━━━━━━━━━━━
⏱️ 合計: 4時間30分
```

### `/week` - 週間サマリー

```
@timebot /week
```

**処理内容**:
1. 今週の作業ログをプロジェクト別に集計
2. 各プロジェクトの作業時間・セッション数と合計を表示

**応答例**:
```
📊 【今週の作業サマリー】

📁 saixaid: 15時間30分（8回）
📁 プロジェクトB: 5時間0分（3回）

━━━━━━━━━━━━━━━━━━
⏱️ 合計: 20時間30分
```

### `/add [プロジェクト名] [時間] [メモ]` - 作業時間追加

```
@timebot /add saixaid 2時間
@timebot /add saixaid 2時間 MTG
```

**対応フォーマット**:
| フォーマット | 例 | 解釈 |
|------------|-----|------|
| 日本語（時間） | `2時間` | 120分 |
| 日本語（分） | `30分` | 30分 |
| 日本語（複合） | `2時間30分` | 150分 |
| 小数 | `1.5時間` | 90分 |
| 英語 | `2h`, `30m`, `2h30m` | 120分, 30分, 150分 |
| 数字のみ | `90` | 90分 |

**応答例**:
```
✅ 作業時間を追加しました
📁 プロジェクト: saixaid
⏱️ 追加時間: 2時間0分
📊 本日の合計: 4時間30分
📝 メモ: MTG
```

### `/undo` - 直近の記録を削除

```
@timebot /undo
```

**処理内容**:
1. 直近の完了済み作業ログを取得
2. 該当レコードを削除
3. 削除したプロジェクト名と時間を表示

**応答例**:
```
🗑️ 直近の記録を削除しました
📁 プロジェクト: saixaid
⏱️ 削除した時間: 2時間0分
```

### `/summary [YYYY-MM]` - 月間サマリー

```
@timebot /summary           # 今月のサマリー
@timebot /summary 2026-01   # 指定月のサマリー
```

**応答例**:
```
📊 【2026年1月の作業サマリー】

📁 saixaid（クライアント名）
   ⏱️ 合計: 45時間30分（15回）
   💰 請求額: ¥86,450

━━━━━━━━━━━━━━━━━━
⏱️ 総作業時間: 45時間30分
💰 総合計: ¥86,450
```

### `/project` - プロジェクト管理

```
@timebot /project              # 一覧表示
@timebot /project add [名前] [時給]  # プロジェクト追加
```

**処理内容（add）**:
1. プロジェクト名の重複チェック
2. Supabaseにプロジェクトを登録
3. 登録結果を表示

**応答例（一覧）**:
```
📁 【プロジェクト一覧】

• saixaid（クライアント名）- ¥5,000/h
• プロジェクトB（株式会社B）- ¥4,500/h
```

**応答例（追加）**:
```
✅ プロジェクトを追加しました
📁 名前: newproject
💰 時給: ¥3,000
🏢 クライアント: クライアント名
```

## 月末自動通知

毎日23:00（JST）に月末かどうかをチェックし、月末の場合は自動で月間サマリーを `SLACK_CHANNEL_ID` に送信します。

**通知例**:
```
📅 本日は締め日です

【2026年1月の作業サマリー】

📁 saixaid（クライアント名）
   ⏱️ 合計: 45時間30分
   💰 請求額: ¥86,450

━━━━━━━━━━━━━━━━━━
⏱️ 総作業時間: 45時間30分
💰 総合計: ¥86,450
```

## エラーハンドリング

### プロジェクトが見つからない場合
```
❌ プロジェクト「XXX」が見つかりません
登録済みプロジェクト: プロジェクトA, プロジェクトB, 保守案件C
```

### 既に作業中の場合
```
⚠️ 現在「プロジェクトA」で作業中です
先に /out で終了してください
```

### 作業中でないのに /out した場合
```
⚠️ 現在作業中ではありません
/in [プロジェクト名] で開始してください
```

## データ構造

### Supabase テーブル

**projects（プロジェクトマスタ）**:
- id: UUID
- name: プロジェクト名
- client_name: クライアント名
- hourly_rate: 時間単価（円）
- is_active: 有効フラグ

**time_logs（作業ログ）**:
- id: UUID
- project_id: プロジェクトID
- started_at: 開始時刻
- ended_at: 終了時刻
- duration_minutes: 経過時間（分）
- note: メモ

## 使用例

### 1日の作業フロー

```
09:00 @timebot /in プロジェクトA
      → ✅ 作業を開始しました

12:00 @timebot /out
      → ✅ 作業を終了しました（3時間0分）

13:00 @timebot /in プロジェクトB
      → ✅ 作業を開始しました

15:30 @timebot /status
      → 🔵 作業中（2時間30分経過）

18:00 @timebot /out
      → ✅ 作業を終了しました（5時間0分）
      → 📊 本日の合計: 8時間0分
```

### 作業時間の後追い追加

```
@timebot /add saixaid 2時間 昨日のMTG忘れてた
→ ✅ 作業時間を追加しました

@timebot /add saixaid 30分
→ ✅ 作業時間を追加しました
```

### 月間サマリーの確認

```
@timebot /summary
→ 📊 【2026年2月の作業サマリー】
  ...

@timebot /summary 2026-01
→ 📊 【2026年1月の作業サマリー】
  ...
```
